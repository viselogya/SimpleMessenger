# SimpleMessenger
SimpleMessenger — это чат-приложение на C++, поддерживающее аутентификацию пользователей, обмен сообщениями и базовую архитектуру клиент-сервер.

Проект состоит из трёх частей:
- **Client** — веб-клиент (в данный момент не реализован, добавлен для полноты структуры).
- **Server** — основная часть проекта. Многопоточный HTTP-сервер на C++ с поддержкой HTTPS.
- **Website** — демонстрационный сайт для сервера. Создан с помощью нейросетей.

## Server

Основная работа велась над сервером.  

### Возможности
- Многопоточный HTTP-сервер (каждое подключение обслуживается отдельным потоком).
- Поддержка HTTPS с самоподписанным сертификатом.
- JWT-аутентификация (через [jwt-cpp](https://github.com/Thalhammer/jwt-cpp)).
- Хранение данных в SQLite.
- Простая маршрутизация HTTP-запросов.
- конфигурация через JSON.
- логирование в текстовый файл

- ## Внешние библиотеки

Проект использует следующие сторонние библиотеки:

- [Boost.Beast](https://www.boost.org/doc/libs/)  
- [Boost.Asio](https://www.boost.org/doc/libs/)  
- [jwt-cpp](https://github.com/Thalhammer/jwt-cpp)  
- [nlohmann/json](https://github.com/nlohmann/json)  
- [OpenSSL](https://www.openssl.org/)  
- [pugixml](https://pugixml.org/)  
- [SQLite3](https://www.sqlite.org/index.html)  

### Установка (Windows, MSVC + vcpkg)
```powershell
vcpkg install boost-beast boost-asio nlohmann-json openssl pugixml sqlite3 jwt-cpp
```

### Запуск сервера
Чтобы запустить сервер локально:

1) Склонируйте репозиторий на свой компьютер (используйте git clone <URL репозитория>).
2) Установите необходимые библиотеки с помощью vcpkg, как описано в разделе "Установка" выше.
3) Откройте файл решения (.sln) в Visual Studio.
4) Соберите проект (Build Solution) и запустите его (Start Debugging или Start Without Debugging).

Сервер запустится и будет слушать подключения по указанному в конфигурации порту (по умолчанию на порту 443).

## Website
Демонстрационный сайт служит для показа возможностей сервера. Он создан с помощью нейросетей и включает базовый интерфейс для аутентификации и обмена сообщениями. Сайт можно использовать как временный клиент для тестирования.

1) Убедитесь, что сервер запущен на вашем локальном устройстве (см. инструкции по запуску сервера выше).
2) Откройте папку с сайтом (директория website или аналогичная) в Visual Studio Code.
3) Установите расширение Live Server из магазина расширений VS Code (если оно ещё не установлено).
4) Запустите Live Server: правой кнопкой мыши кликните на основной HTML-файл (например, index.html) и выберите "Open with Live Server".

Это откроет сайт в браузере по адресу вроде http://127.0.0.1:5500/. Теперь вы можете тестировать взаимодействие с сервером.
